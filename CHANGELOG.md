# 更新日志

## 0.1.29

- 修复内嵌面板点击“归档”时因本机找不到 `codex` 命令导致 `spawn codex ENOENT` 的问题，CLI 失败后会安全回退为移动本地会话文件。
- 删除按钮现在会在每条会话卡片中始终显示；普通会话保持禁用并提示需先归档，已归档会话仍保留二次确认后删除。
- 为内嵌面板搜索框、顶部按钮和会话卡片操作按钮补充中英文悬浮说明，方便区分每个入口的作用。

## 0.1.28

- 调整侧栏视图顺序：“会话”树默认排在“搜索与编辑”面板上方。
- 将首次加载默认过滤改为“本工作区”，让侧栏优先聚焦当前项目；设置项说明同步中英文。
- 在“搜索与编辑”结果卡片中新增归档、取消归档和删除入口；删除继续仅允许已归档会话，并保留二次确认。

## 0.1.27

- 新增侧栏“搜索与编辑”面板，提供真正嵌入列表区域的搜索输入框，不再依赖 VS Code 顶部 QuickPick 搜索框。
- 搜索输入会实时同步过滤会话树，继续优先使用内存快照和磁盘缓存，避免搜索前重复全量加载会话。
- 重命名、项目标签、备注改为在侧栏面板中行内编辑，减少顶部输入框打断当前操作流。

## 0.1.26

- 修复 v0.1.25 后桌面端“无工作区”会话被历史工作区根路径重新归入“本工作区 / 其他工作区”的回归问题。
- 新增读取桌面端 `.codex-global-state.json` 中的 `projectless-thread-ids`，并以该标记作为会话归组的最高优先级。
- 磁盘缓存恢复时也会重新校准“无工作区”标记，避免旧缓存导致刷新后分组仍不正确。

## 0.1.25

- 修复 VS Code 配置变更后数据源仍沿用旧 `codexHomeOverride` / `codexCliPath` 的问题，切换 CODEX_HOME 或 CLI 路径后无需重载窗口即可刷新生效。
- 会话归组会额外参考桌面端保存的 `active-workspace-roots` 与 `electron-saved-workspace-roots`，减少真实工作区会话掉入“无工作区”的情况。
- 优化桌面端置顶会话在插件右键“置顶/取消置顶”时的反馈：桌面端置顶需在桌面端取消，插件本地置顶与桌面端置顶会分开提示。

## 0.1.24

- 同步 Codex 桌面端 `.codex-global-state.json` 中的 `pinned-thread-ids`，桌面端置顶会话会在插件里显示置顶标识并参与置顶排序。
- 桌面端取消置顶后，插件刷新时会自动恢复普通排序，避免曾经置顶过的会话状态残留或显示异常。
- 桌面端置顶与插件本地置顶分开记录：插件只读取桌面端状态，不写入桌面端状态文件，降低两边互相覆盖的风险。

## 0.1.23

- 修复“无工作区”分组误把已知项目会话拆散的问题：若会话路径能命中已有工作区根路径，即使缺少单条 hint，也会自动并回对应项目分组。
- “无工作区”与“未归类”不再继续按路径尾名伪造项目文件夹，改为直接显示“会话 / 已归档”子层，避免出现看起来像重复工作区的目录。
- “无工作区”分组默认折叠，减少侧栏干扰。

## 0.1.22

- 项目下固定新增“会话”子文件夹，普通会话始终放在该层级内，避免隐藏归档后 VS Code TreeView 将叶子节点图标列向左收缩。
- “已归档”继续作为同项目下的并列子文件夹展示，显示/隐藏归档时整体层级更稳定。
- 新增“无工作区”分组，将只有路径但未真正归属到工作区的会话单独展示，不再混入“其他工作区”。

## 0.1.21

- 修复切换显示归档后树视图会话行缩进视觉不稳定的问题，改用更稳定的缩进字符。
- 将“切换归档”命令文案改为“显示/隐藏归档会话”，避免与单个会话的归档操作混淆。

## 0.1.20

- 移除 `onStartupFinished` 激活事件，避免 VS Code 启动阶段过早触发官方 Codex Webview。
- 保留打开会话管理视图和执行命令时的按需激活，降低与官方 Codex 插件 Webview 初始化的耦合。

## 0.1.19

- 新增会话快照 JSON 磁盘缓存，存放在 VS Code 扩展 `globalStorage`，首次进入视图会先展示上次缓存，再后台刷新真实数据并覆盖缓存。
- 搜索并打开会话优先使用当前内存快照或磁盘缓存，避免每次搜索都重新全量拉取会话记录。
- 本工作区、全部工作区、归档显示和旧过滤命令改为基于当前快照即时重算，减少无意义的全量刷新。
- 新增 `codexSessionManager.enableDiskCache` 配置项，默认开启，可按需关闭缓存。
- 收敛侧边栏标题栏搜索入口，只保留“搜索并打开会话”，避免两个搜索入口混淆。

## 0.1.18

- 修复首次点击其他工作区会话时“正在切换”通知阻塞实际切换的问题，点击后会立即切换工作区。
- 工作区切换后会在扩展启动时自动恢复待打开会话，并对官方 Codex 会话打开增加短重试，减少需要重复点击的问题。
- 新增“搜索并打开会话”入口，可按标题、预览、备注、路径、项目、分支和会话 ID 搜索所有工作区与归档会话。
- 搜索过滤改为插件本地统一过滤，避免 app-server 预过滤导致本地别名、备注、项目标签无法搜到。
